# O - Open Close Principle (OCP)

## Definition

Software entities (classes, modules, functions) should be **open for extension** but **closed for modification**.

## Simple terms

- **Open for extension** = You can add new features easily.
- **Closed for modification** = Don't change existing working code.

## Example

Adding new payment methods (**Credit Card, UPI, PayPal**) without modifying the existing payment processing code.
